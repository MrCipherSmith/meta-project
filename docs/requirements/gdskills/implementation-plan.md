# gdskills Implementation Plan

Version: 0.1.0

## Goal

Implement the first production slice of the native bundled `gdskills` module.

The module must be self-contained inside the `gd-metapro` package. It must not require `goodai-base`, `~/goodai-base`, or any external skill repository.

## Scope

This implementation creates the foundation for bundled working skills:

- native bundled skill catalog in TypeScript;
- install profiles: `minimal`, `recommended`, `full`, `custom`;
- installation into `.metaproject/skills/gdskills/`;
- generated `.metaproject/skills/catalog.md`;
- module manifest `.metaproject/modules/gdskills.md`;
- CLI commands for install/status/catalog;
- `gd-metapro init` integration;
- local-first agent entrypoint references.

## Phase 3: CLI Commands

Actions:

- register `gd-metapro skills` in `src/cli.ts`;
- support:
  - `gd-metapro skills status`;
  - `gd-metapro skills catalog [--profile <profile>]`;
  - `gd-metapro skills install [--profile <profile>]`.

Verification:

## Non-goals For This Slice

- implement `skills generate`, `skills verify`, `skills learn`, `skills export`, `skills sync`;
- create fully specialized review logic implementations;
- integrate real multi-agent runtime execution;
- sync to global Codex/Claude skill folders by default;
- depend on external skill repositories.

## Guardrails

- Keep generated skills concise and procedural.
- Keep `SKILL.md` valid for Codex/Claude-style skill loading: frontmatter with `name` and `description`.
- Do not create README/changelog clutter inside runtime skill folders.
- Do not overwrite user-authored files unless they are generated by this module.
- Preserve local-first routing: `.metaproject/index.md`, `.metaproject/skills/catalog.md`, project-skills, gdskills, optional global fallback.
