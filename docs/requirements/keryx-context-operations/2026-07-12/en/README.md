# Keryx Context Operations
Version: 1.0.0

## Status

`specification ready — future implementation`. This is the synchronized English
view of the canonical Russian package; requirement IDs (`CO-*`) are normative.

## Purpose

Context Operations is a future Git-native, governed context assembly layer for
coding agents. It composes bounded, cited project context from code graph, wiki,
memory, rules, skills and quality evidence without replacing Keryx’s
deterministic local-first core.

## Non-goals

No required cloud database, no new agent runtime, no automatic promotion of
untrusted content, and no replacement for Mem0, Letta, Graphiti, Cognee or
OpenViking.
