# Review Learning
Version: 1.0.0

- Documentation readiness requires executable evidence, not a statement that
  a validator is planned. The final package records parser, schema, fixture,
  and semantic commands with deterministic output.
- A migration-only schema must be absent from the active fixture universe;
  a deprecation flag alone leaves a second source of truth.
- Coordinator ownership is clearest when both the positive authority and the
  forbidden imports/write paths are named. The `execution/turn-control` text
  now provides that boundary.
- Compatibility policy is reviewable only when ranges, migration IDs, and
  rejection behavior are machine-readable, so the registry is part of the
  package rather than prose-only job context.
