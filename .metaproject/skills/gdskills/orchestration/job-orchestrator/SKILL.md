# job-orchestrator

## Purpose

Run full task pipelines: clarify, collect context, plan, implement, verify, review, and summarize.

## When To Use

- implement issue
- full workflow
- orchestrate task

## Workflow

1. Clarify ambiguity with interviewer-style questions only when required.
2. Collect compact context through Metaproject modules before implementation.
3. Break the work into phases with verification after each phase.
4. Run review and skill-learning handoffs before final summary when relevant.

## Local-First Rules

1. Start from `.metaproject/index.md` and `.metaproject/skills/catalog.md`.
2. Prefer project-local skills under `.metaproject/project-skills` and `.metaproject/skills/gdskills`.
3. Use `gdgraph`, `gdctx`, `gdwiki`, Code Health, and Documentation Memory when they provide narrower context.
4. Treat external/global skills only as explicit fallback when local Metaproject does not provide the capability.
5. Verify conclusions against source files before reporting or editing.
