# Acceptance Criteria

Rules:

- Criteria lines use the exact format `- ACn: <criterion>`.
- After `flow freeze` this file is checksum-protected: any edit outside
  `keryx flow ac update` fails every gate and status transition.
- Completion requires every ACn to be confirmed via
  `keryx flow ac confirm <id> <ACn>`.

## Criteria

- AC1: RP-01 ProviderPort conformance + normalization — `src/harness/provider/anthropic/anthropic-provider.ts` defines `AnthropicProvider implements ProviderPort` (W5) with `describe()` and `stream(request, opts)`; a recorded Anthropic Messages-API SSE transcript replayed through `stream()` yields the exact `NormalizedEventKind` sequence (model_start / text_delta / tool_call_start / tool_call_delta / tool_call_end / usage_update / model_end) with the correct per-kind fields (`text`/`toolName`/`toolCallId`/`inputDelta`/`input`/`usage`), monotonic `sequence`, and `attemptId === opts.attemptId`; NO SDK or provider-wire type crosses the `ProviderPort` boundary (thin `fetch`/SSE only); `opts.signal` cancels the in-flight attempt. (Schema note, mirroring the W6 FakeProvider precedent: a bare in-memory `NormalizedEvent` is intentionally NOT the durable `harness-event.schema.json` envelope — `assertEventValid` correctly REJECTS the bare event; the durable envelope is constructed downstream, so AC1 asserts the correct normalized shape/sequence rather than direct envelope validation.)
- AC2: Storage-off / privacy-retention contract — the adapter exposes a durable descriptor document (e.g. `descriptorDocument()`) whose `remoteState.storage`/`retention`/`continuation` are `false` and which VALIDATES against the frozen `provider-descriptor.schema.json` (`const:false`), while `describe()` returns the W5 in-memory `ProviderDescription` with matching capabilities; the adapter persists no prompt/response by default (no persistence path without an explicit opt-in); the credential (API key) is never persisted or logged and is redacted from every `NormalizedError.message` — via a deterministic exact-match scrub of the known key (equivalent-or-stronger than the W7 heuristic redactor for a known secret; the W7 redactor remains available for general content), proven by a test that injects the key into a provider error body and asserts it is stripped. The Anthropic Messages API is used statelessly (no server-side storage/retention/continuation).
- AC3: Capability gate + guarded egress (fail-closed) — the live `fetch` fires ONLY when an explicit capability grant (network + credential) is present; with no grant the adapter returns a fail-closed `NormalizedError` and performs NO network call (the mocked `fetch` is never invoked); the base-URL host is checked with the reused W15 private-egress predicate, so a private / loopback / link-local / metadata destination is denied fail-closed; a public host (`api.anthropic.com`) is permitted.
- AC4: Provider negatives → fail-closed (9-kind taxonomy) — timeout, cancellation (AbortSignal → `cancelled`), malformed/torn SSE (`malformed`), truncated stream, rate-limit (401→`authentication`, 400→`invalid_request`, 429→`rate_limit` retryable with `retryAfterMs`, 529→`overloaded`, 5xx→`unavailable`), and egress-deny / missing-capability each map to the correct `ProviderErrorKind` as a `NormalizedError`/`provider_error`, with `retryable` set per `defaultRetryable`; no negative yields a partial-but-“complete” attempt (fail-closed).
- AC5: No regression / offline-determinism / scope / deps — `tsc --noEmit` is clean and the full `bun test` suite is ≥ the pre-change baseline of 1114 pass with the new tests green and 0 fail, and the ENTIRE suite runs OFFLINE (no live network: all provider tests replay recorded transcripts / mock `fetch`; the real call is behind a capability flag and never runs in CI); behavior is deterministic (no `Date.now`/`Math.random`; injected clock/id); no new production dependency — `dependencies` REMAINS `{}` (thin `fetch`, no SDK); new code lives under `src/harness/provider/anthropic/` only; the W5 ProviderPort/FakeProvider, W6 fixtures, W7 redaction, and W15 egress guard are REUSED (a minimal additive `export` of the W15 egress predicate is the only permitted change to a prior module — no behavior change); the adapter never writes flow.json (D-02); the frozen requirements package, canonical contract schemas, `src/eval/`, `src/contracts/`, and ADR-0001…0004 are NOT modified.
