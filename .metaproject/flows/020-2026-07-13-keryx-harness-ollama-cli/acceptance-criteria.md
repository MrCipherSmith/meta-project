# Acceptance Criteria

Rules:

- Criteria lines use the exact format `- ACn: <criterion>`.
- After `flow freeze` this file is checksum-protected: any edit outside
  `keryx flow ac update` fails every gate and status transition.
- Completion requires every ACn to be confirmed via
  `keryx flow ac confirm <id> <ACn>`.

## Criteria

- AC1: Ollama ProviderPort adapter — `src/harness/provider/ollama/ollama-provider.ts` defines `OllamaProvider implements ProviderPort` over the OpenAI-compatible `/v1/chat/completions` SSE endpoint via a thin injected `fetch` (NO SDK); a recorded transcript replayed through `stream()` yields the exact `NormalizedEventKind` sequence (model_start / text_delta / tool_call_start / tool_call_delta / tool_call_end / usage_update / model_end) with correct fields; `opts.signal` cancels; NO SDK or provider-wire type crosses the `ProviderPort` boundary. `describe()` exposes a descriptor with storage/retention/continuation = false (local, stateless).
- AC2: Narrow loopback egress opt-in (SECURITY — no SSRF weakening) — an additive `isLoopbackHost(host)` predicate in `src/harness/mutation/guard.ts` returns TRUE only for loopback forms (127.0.0.0/8, `::1`/`[::1]`/IPv4-mapped loopback, `localhost`, and the encoded loopback forms the existing decoder recognizes) and FALSE for metadata (169.254.0.0/16), link-local, and private-LAN (10/8, 172.16–31, 192.168/16) and public hosts; the Ollama adapter permits egress IFF `!isPrivateEgressHost(host)` OR (`grant.allowLoopback === true` AND `isLoopbackHost(host)`). Consequently: a loopback base URL is DENIED (fail-closed) WITHOUT `allowLoopback`; a metadata/link-local/private-LAN base URL is DENIED even WITH `allowLoopback`; a public host is unaffected. The W15 guard's general fail-closed behavior and the W14 Anthropic adapter's default egress are UNCHANGED (all prior guard/adapter tests stay green).
- AC3: Provider negatives fail-closed + credential/no-leak — the Ollama adapter maps provider negatives to the 9-kind taxonomy (404 model-not-found → invalid_request; connection-refused / 5xx → unavailable; malformed / torn SSE → malformed; AbortSignal → cancelled), each a fail-closed `provider_error`/`NormalizedError` with `retryable` per `defaultRetryable`, no spurious `model_end`; no secret/credential material leaks into any event or error message.
- AC4: `keryx harness run` CLI — `src/commands/harness.ts` (registered in `src/cli.ts`) provides `keryx harness run --provider <fake|anthropic|ollama> --model <m> [--base-url <url>] "<prompt>"` that assembles `runOffline` with real deps and the selected provider and prints the normalized events / final text / completion / evidence; `fake` runs fully offline (deterministic default, no network); `ollama` uses the loopback opt-in grant; `anthropic` reads `ANTHROPIC_API_KEY` from env and, when absent, returns a clear fail-closed message with NO network call. The CLI never writes flow.json (D-02). An OFFLINE test exercises the `fake` path (no live network).
- AC5: No regression / offline-determinism / scope / deps — `tsc --noEmit` is clean and the full `bun test` suite is ≥ the pre-change baseline of 1160 pass with the new tests green and 0 fail, and the ENTIRE automated suite runs OFFLINE (recorded transcripts / mocked `fetch`; the live smoke is a separate manual orchestrator step, not in CI); behavior in tests is deterministic (no `Date.now`/`Math.random`); no new production dependency — `dependencies` REMAINS `{}` (thin `fetch`, no SDK); new code lives under `src/harness/provider/ollama/` + `src/commands/harness.ts`, and the only changes to prior modules are additive (the `isLoopbackHost` export + the adapter opt-in branch + the `src/cli.ts` command registration); the W5 ProviderPort/FakeProvider, W6 fixtures, W7 redaction, and W14 Anthropic adapter are REUSED not rewritten; the adapter/CLI never write flow.json; the frozen requirements package, canonical contract schemas, `src/eval/`, `src/contracts/`, and ADR-0001…0004 are NOT modified. A live smoke run of `keryx harness run --provider ollama --model llama3.1:latest` against the local server succeeds end-to-end (recorded in the flow journal; not a CI gate).
